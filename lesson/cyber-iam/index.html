Cybersecurity

Identity and Access Management (IAM)

Définition: IAM regroupe les politiques et technologies qui garantissent que les bonnes personnes ont le bon accès.

Kreyòl: IAM kontwole kiyès ki ka antre nan sistèm nan ak sa yo ka fè.

Why it matters

Most breaches involve credential misuse or privilege abuse.

Core concepts

  • MFA
  • RBAC
  • Least privilege
  • Access reviews

Deep explanation

Modern IAM combines MFA, SSO, RBAC, lifecycle provisioning, and continuous verification.

Step-by-step breakdown

  1. Create role matrix.
  2. Enable MFA.
  3. Automate onboarding/offboarding.
  4. Audit privileged access.

Code and commands (English)

# Linux: list users with shell access
awk -F: '$7 ~ /bash|zsh/ {print $1}' /etc/passwd

Erreurs fréquentes

  • Shared admin accounts
  • No offboarding automation

Practical exercises

  • Design roles for SOC analyst, engineer, and manager.

Challenge exercises

  • Write an IAM policy for a cloud startup with contractors.

Mini quiz

  • What is the difference between authentication and authorization?

Summary

IAM reduces lateral movement and limits blast radius.
